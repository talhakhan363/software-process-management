Objective: Define the foundational steps required before the actual coding phase begins.

Key Phases:
Feasibility Analysis: Evaluating the project through Technical, Operational, and Economic lenses to ensure viability.

Stakeholder Identification: Mapping out primary users (students/faculty) and secondary stakeholders (administration) to align project goals.

Requirement Elicitation: Drafting the initial Software Requirement Specification (SRS) including both functional and non-functional requirements.

SDLC Selection: Choosing the appropriate methodology (e.g., Agile for flexibility or Waterfall for fixed requirements).
